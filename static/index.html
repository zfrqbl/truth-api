Truth API

Simple, secure API for random truth prompts. MIT Licensed.

Overview

Truth API provides random truth prompts for games, apps, and social experiences. It's fast, secure, and easy to integrate.

Open Swagger Documentation

API Endpoints

Health Check

GET /health
Check if the API is running and healthy.
Example Response: { "status": "healthy", "version": "1.0.0", "data_loaded": true }

List Categories

GET /categories
Get all available truth prompt categories.
Example Response: ["default", "spicy", "family"]

Get Statistics

GET /stats
Get statistics about loaded truth prompts per category.
Example Response: { "default": { "truth_count": 50, "total_count": 50 } }

Get Random Truth

GET /random
Get a random truth prompt. Optionally filter by category.
category Category name (default: "default") optional
Basic Request: curl https://truth-api.up.railway.app/random
With Category: curl https://truth-api.up.railway.app/random?category=spicy
Example Response: { "type": "truth", "text": "What's the most embarrassing thing you've ever done?", "category": "default" }

Reload Data (Admin)

GET /reload
Reload truth prompt data from files. Note: This endpoint should be protected in production environments.

Error Responses

The API returns standard HTTP status codes with JSON error messages.

Status Code Error Type Description
400 Bad Request Invalid category name or malformed input
404 Not Found Category not found or no prompts available
404 Not Found Endpoint does not exist
422 Validation Error Request validation failed (FastAPI validation
